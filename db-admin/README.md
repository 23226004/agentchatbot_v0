# db-admin

DB에 쌓인 데이터를 **확인하고 관리**하는 공간. 추후 완전 자동화를 위해
미리 분리해 둔 계층이다. (`database/`의 스토리지 정의와 분리)

## 구조 (`src/`)
| 폴더 | 책임 |
|------|------|
| `inspector/` | 데이터 조회·검증·품질 점검 |
| `pipeline/` | 적재·정제·동기화 파이프라인 |
| `automation/` | 스케줄링·자동 관리 작업 (향후 확장) |

## 원칙
- 운영용 관리 계층으로, 서비스 런타임(backend/agent)과 분리한다.
- DB 접근은 backend의 추상 인터페이스를 재사용한다.
